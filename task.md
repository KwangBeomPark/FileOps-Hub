# 프로젝트 전수 점검 작업 계획

## 목표
- 팀별 자료 폴더와 공용 배포 폴더를 안전하게 동기화한다.
- PDF/EML 이미지 변환, OCR, Office/PDF 포맷 변환을 일괄 실행한다.
- 앱 실행 중 지정 시각에 일괄 작업을 자동 실행하고 결과를 이메일 또는 로컬 보고서로 남긴다.
- 실제 구현, 테스트, 배포 설정과 프로젝트 문서를 일치시킨다.

## Phase
- [x] Phase 1: 필수 지침, 소스 구조, 설정, 테스트 체계 감사
- [x] Phase 2: 기존 34개 회귀 테스트, 컴파일, 정적 검사, 빌드 검사 실행
- [x] Phase 3: 동기화 충돌 처리 및 통합 실행 결과 판정 수정
- [x] Phase 4: 일일 예약 실행과 이메일 결과 통지 보강
- [x] Phase 5: 누락 테스트, 설정 기본값, 배포 의존성 보강
- [x] Phase 6: GUI/빌드/전체 회귀 검증 및 문서 최신화

## 주요 리스크
- 네트워크 드라이브와 동기화 폴더는 파일 잠금, 지연, 동일 시각 수정 충돌이 발생할 수 있다.
- Office 변환은 Windows에 설치된 Excel/Word/PowerPoint COM 자동화에 의존한다.
- 예약 실행은 앱이 실행 중일 때만 동작하며 Windows 서비스나 작업 스케줄러를 대체하지 않는다.
- 실제 SMTP 발송은 사용자 계정과 회사 네트워크가 필요하므로 자동 테스트에서는 프로토콜을 모킹한다.
